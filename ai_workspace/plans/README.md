# 执行方案存储目录

## 目录说明

本目录用于存储由 `/discuss` 命令生成的执行方案。

## 方案文件格式

### 文件命名

格式：`PLAN-[话题ID]-[序号].md`

示例：
- `PLAN-C-001.md` - 话题 C 的第 1 个方案
- `PLAN-A-002.md` - 话题 A 的第 2 个方案

### 方案文件结构

```markdown
# 执行方案 PLAN-[话题ID]-[序号]

## 方案信息
- **方案编号**: PLAN-[话题ID]-[序号]
- **关联话题**: [话题ID] - [话题名称]
- **创建时间**: YYYY-MM-DD HH:MM:SS
- **创建者**: /discuss
- **状态**: 待执行 / 执行中 / 已完成 / 已取消

## 方案目标
[方案的核心目标]

## 执行步骤
1. [步骤1描述]
2. [步骤2描述]
3. [步骤3描述]
...

## 相关文件
- [文件路径1]
- [文件路径2]
...

## 预期结果
[预期达到的结果]

## 注意事项
[需要特别注意的事项]

## 执行记录
- [执行时间] - [执行者] - [执行结果]
```

## 方案索引

所有方案都会在 `INDEX.md` 中建立索引，方便快速查找。

## 使用流程

1. **生成方案**：使用 `/discuss` 讨论后，用户说"可以生成方案了"，discuss 会生成方案文件
2. **执行方案**：使用 `/crew PLAN-XXX-XXX` 或 `/crew`（自动识别）来执行方案
3. **更新状态**：执行后，方案状态会自动更新

## 维护说明

- 方案文件由 `/discuss` 命令创建
- 方案状态由 `/crew` 或 `/prompt_engineer` 命令更新
- 索引文件由相关命令自动维护
